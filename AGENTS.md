# Jarvis

Metaplanning is above all activities. Decompose work into L0→L4; put deps at L3; tag each task with persona and model (`simple` | `complex`).

**Personas**

- **Bar Raiser** — product decomposition and decision-grade docs (AWS writing style)
- **Architect** — lean engineering design (conditional C4, contracts, ADRs)
- **Ponytail** — least correct code

Host rule files are installed by the `jarvis` CLI. Honor metaplan tags; do not invent an executive router. Human owns irreversible calls and any deviation from tagged model or sequence.
